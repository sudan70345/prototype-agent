# Pencil 执行 + 自校验闭环 Skill — 阶段 3

## Critical 级约束（必须遵守）

### 文件架构约束
1. **一任务只创建一个 .pen 文件**，命名为 `{task_name}.pen`
2. **主页面和弹窗均作为该文件内的独立 Pencil 页面**，通过 Pencil 的「页面（Pages）」功能管理
3. **禁止为每个主页面或弹窗单独创建 .pen 文件**
4. **页面命名规范**：主页面 `P{N}_{页面名}` / 弹窗页面 `P{N}D{M}_{弹窗名}`

### Pencil 兼容性约束
5. **Pencil 桌面应用必须已运行**
6. **创建 .pen 文件后先提示用户 Cmd+S 手动保存**
7. **单次 batch_design 操作数 ≤ 15**
8. **全部颜色使用 #RRGGBB 十六进制格式**
9. **fontFamily 使用字符串字面量**
10. **文件路径使用相对路径**

### 弹窗页面约束
11. **弹窗作为 .pen 文件内的独立 Pencil 页面**，禁止将弹窗内容叠加绘制在主页面画布上
12. **弹窗页面画布尺寸**：确认弹窗 400×180，表单弹窗 400×240～800×520（按内容量设定）
13. **弹窗页面布局**：标题左上角 + 内容区居中 + 按钮右下角（或底部居中）

### 视觉质量约束
14. **行内容宽度**：文本/输入框/按钮不得超出所在容器宽度，预留左右各 ≥16px padding
15. **模块间距**：相邻模块之间保持 ≥12px 间距，避免内容紧贴或重叠
16. **列对齐**：表格/列表的同列元素必须左对齐或右对齐一致，禁止参差不齐
17. **操作文字对齐**：同行操作链接/按钮文字必须基线对齐
18. **高宽防遮挡**：容器高度必须能完整容纳内部所有内容，禁止内容被裁切；宽度不足时优先换行而非缩小字号
19. **固定元素不遮挡**：顶部导航栏、侧边栏等固定元素不得遮挡内容区主体

## 工作流程

## 注意：何时调用 Pencil 执行 Skill（新增）

此 `pencil-executor` Skill 仅在以下情况下被触发：

- 用户在初始调用决策中明确同意进行原型设计（对于全新需求）；或
- 用户指定了 `prototype/{task_name}` 并明确要求更新/维护现有 `.pen`（包括在原有 `.pen` 中新增页面或修改页面）。

若上述条件未满足，agent 不应调用本 Skill，流程将在阶段2 停止或仅更新规格文档。

### Step 1：健康检查

确认 Pencil MCP 可用，否则提示用户并停止。

---

### Step 2：准备 UI 设计要素

检查任务目录中是否有 `ui-design-elements.md`：
- **有** → 直接使用
- **没有** → 读取 `templates/ui-design-elements.md` 模板，生成默认版本写入任务目录

---

### Step 3：提取完整页面清单 ★ Critical

**在开始绘制任何内容之前**，先扫描 `02-page-specs.md`，提取所有主页面和弹窗，生成完整的绘制清单，写入 `pencil-execution-log.md`。

#### 主页面清单

| # | Pencil 页面名 | 对应规格章节 | 状态 |
|---|-------------|------------|------|
| P1 | P1_{页面名} | 页面[1] | 待绘制 |
| P2 | P2_{页面名} | 页面[2] | 待绘制 |

#### 弹窗页面清单

扫描每个主页面的「弹窗」「交互定义」章节，提取所有弹窗：

| # | Pencil 页面名 | 所属主页面 | 弹窗类型 | 状态 |
|---|-------------|---------|---------|------|
| P1D1 | P1D1_{弹窗名} | P1_{主页面名} | 表单弹窗 | 待绘制 |
| P1D2 | P1D2_{弹窗名} | P1_{主页面名} | 确认弹窗 | 待绘制 |

> ⚠️ 弹窗清单**不可遗漏**。发现规格中有弹窗未加入清单时，立即补入。

---

### Step 4：创建 .pen 文件

1. 创建 `prototype/{task_dir}/{task_name}.pen`
2. ⚠️ **提示用户立即 Cmd+S 保存**，保存前不得进行任何绘制操作

---

### Step 5：逐页面绘制（主页面）

按 Step 3 主页面清单顺序，在 .pen 文件内逐一新增 Pencil 页面并绘制：

#### 5.1 新增 Pencil 页面
- 在 .pen 文件内新增页面，命名为 `P{N}_{页面名}`

#### 5.2 分批 batch_design 绘制
- 拆解页面组件，每批 ≤ 15 个操作
- 引用 `ui-design-elements.md` 中的颜色、字体、间距等设计令牌
- **绘制时严格遵守视觉质量约束**（见 Critical 约束 14-19）

#### 5.3 视觉质量自检（绘制完每个主页面后立即执行）
在提交截图前，逐项检查：
- [ ] 各行文本/输入框/按钮是否超出容器宽度？（左右各 ≥16px padding）
- [ ] 相邻模块之间是否有 ≥12px 间距？有无内容紧贴或重叠？
- [ ] 表格/列表各列是否对齐一致？
- [ ] 同行操作链接/按钮是否基线对齐？
- [ ] 容器高度是否完整容纳了内部所有内容？有无被裁切？
- [ ] 顶部导航栏/侧边栏是否遮挡了内容区主体？
发现问题 → 立即调用 batch_design 修正，再继续截图

#### 5.4 截图
- 截图保存到 `screenshots/P{N}_{页面名}.png`

#### 5.5 更新清单状态
- 将该页面状态更新为「✅ 已完成」

---

### Step 6：逐页面绘制（弹窗页面）★ Critical

所有主页面绘制完成后，按 Step 3 弹窗清单顺序，在同一 .pen 文件内新增弹窗 Pencil 页面：

#### 6.1 新增弹窗 Pencil 页面
- 在同一 .pen 文件内新增页面，命名为 `P{N}D{M}_{弹窗名}`
- **⚠️ 禁止将弹窗内容绘制在主页面画布上**

#### 6.2 设置弹窗画布尺寸
- 确认/操作弹窗：400×180
- 简单表单弹窗（≤5字段）：400×300～520
- 复杂表单弹窗（>5字段）：600×400～800×520
- 根据实际内容量合理调整

#### 6.3 绘制弹窗内容
布局规范：
- **标题**：左上角，15px bold，与边缘保持 16px 间距
- **内容区**：标题下方，左右各 ≥16px 内边距，文案使用 `textGrowth: "fixed-width"` 自动折行
- **按钮区**：底部右对齐（确认/取消）或底部居中（单按钮）

#### 6.4 视觉质量自检（同 5.3）

#### 6.5 截图
- 截图保存到 `screenshots/P{N}D{M}_{弹窗名}.png`

#### 6.6 更新清单状态
- 将该弹窗状态更新为「✅ 已完成」

---

### Step 7：自校验闭环 ★

所有主页面和弹窗页面绘制完成后，进入自校验阶段。

#### 7.0 完整性检查（必做，不可跳过）

对照 Step 3 的完整清单：
1. 确认每个主页面已有对应截图
2. 确认每个弹窗页面已有对应截图，且为独立 Pencil 页面（非嵌入主页面）
3. 发现遗漏 → 立即补绘，不得进入后续自校验

```
完整性检查结果:
  主页面: ✅ N/N 已完成
  弹窗页面: ✅ M/M 已完成 / ⚠️ 缺失: [列出缺失项]
```

#### 7.1 规格对比（主页面 + 弹窗页面）

对每个页面（包含弹窗页面）：
1. 查看截图
2. 对照 `02-page-specs.md` 中对应的规格描述
3. 逐项评估偏差类型：
   - **布局偏差**：模块位置、排列方式与规格不一致
   - **内容偏差**：模块内容、元素与规格不一致
   - **样式偏差**：颜色、字体、间距等与 UI 设计要素不一致
   - **交互缺失**：规格中定义的交互元素未体现在设计中
   - **视觉质量问题**：行溢出、模块重叠、列错位、按钮错位、内容被裁切、遮挡

#### 7.2 有偏差 → 自动修正

1. 记录偏差详情
2. 调用 batch_design 修正（最多 3 轮）
3. 修正后截图，保存到 `screenshots-reviewed/P{N}_{页面名}_v{轮次}.png`
4. 重新对比；3 轮后仍有偏差 → 标记「⚠️ 需人工调整」

#### 7.3 无偏差 → 标记通过

```
✅ {页面名}: 自校验通过
```

## 错误处理

- 单次 batch_design 失败：重试 2 次
- 截图失败：记录警告，继续
- Pencil MCP 断开：停止，提示用户
