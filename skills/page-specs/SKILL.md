# 逐页沟通 Skill — 阶段 2

## 输入

- `prototype/{task_name}_{YYYYMMDD}/01-business-architecture.md`（已确认）

## 前置条件

- 业务架构文档已通过用户确认

## 输出

- `prototype/{task_name}_{YYYYMMDD}/02-page-specs.md`（累积追加写入）

---

## 工作流程

### Step 1：读取业务架构文档中的页面清单

从 `01-business-architecture.md` 中提取页面清单，确认沟通顺序（通常按页面流转顺序，从首页开始）。

---

### Step 2：逐页面沟通

对每个核心页面按以下顺序逐一与用户对话：

#### 2a. 结构布局
```
「{页面名称} 页面的整体布局是什么样？
 - 上下结构 / 左右分栏 / 卡片流 / 列表 / 网格 / 混合
 - 是否有固定顶部导航栏、底部 Tab Bar、侧边栏？」
```

#### 2b. 模块内容
```
「页面包含哪些功能模块？
 哪些是重要的核心模块？
 哪些是次要/辅助模块？」
```

#### 2c. 关键元素与交互
```
「页面中最重要的交互元素是什么？
 是否有弹窗、浮层、子页面等交互形式？
 用户的操作行为有哪些？（点击、滑动、输入、长按…）」
```

#### 2d. 写入设计描述

每页沟通确认后，将其描述追加到 `02-page-specs.md` 中：

```markdown
## 页面[N]: [页面名称]
> Pencil 页面名: P{N}_{页面名}

### 布局结构
- 整体布局: [布局类型]
- 固定元素: [导航栏 / Tab Bar / 侧边栏…]
- 滚动方式: [纵向 / 横向 / 全屏…]

### 模块清单（按优先级排列）

#### 🔴 重要模块
1. **[模块1名称]**
   - 类型: navigation_bar / banner / grid / list / card / form / chart / table / …
   - 内容: [描述模块具体内容]
   - 交互: [点击跳转 / 展开收起 / 输入 / 滑动…]
   - 设计要点: [颜色、尺寸、间距等特别要求]

2. **[模块2名称]**
   - ...

#### 🟡 次要模块
1. **[模块N名称]**
   - ...

### 关键元素
- [元素1]: [描述，如: 搜索输入框，固定顶部，宽度100%]
- [元素2]: [描述，如: 悬浮按钮，右下角位置，48×48]

### 交互定义
| 触发元素 | 交互类型 | 目标/反馈 | 说明 |
|---------|---------|----------|------|
| 按钮X | click | 跳转至页面Y | push动效 |
| 列表项 | click | 跳转至详情 | push动效 |
| ... | ... | ... | ... |

### 弹窗详细设计
> ⚠️ 每个弹窗在阶段3将作为独立的 Pencil 页面存在于 .pen 文件内，命名为 P{N}D{M}_{弹窗名}
> ⚠️ 禁止将弹窗内容叠加绘制在主页面画布上

#### 弹窗[1]: [弹窗名称]
- Pencil 页面名: P{N}D1_{弹窗名}
- 触发方式: [点击「新增」按钮 / 点击「删除」图标 / …]
- 弹窗类型: 表单弹窗 / 确认弹窗 / 信息展示弹窗
- 画布尺寸: 宽×高（确认弹窗 400×180，表单弹窗根据字段数量设定）
- 标题: [弹窗标题文案]
- 内容区:
  - [字段1]: 类型（输入框/下拉/单选/…），是否必填，占位文案
  - [字段2]: ...
  - [说明文案]: [文案内容]（适用于确认弹窗）
- 按钮区: [按钮1文案]（主操作）/ [按钮2文案]（取消）

#### 弹窗[2]: [弹窗名称]
- ...
```

---

### Step 3：向用户展示每页摘要确认

每页写入后，向用户展示确认摘要：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 {页面名称} — 设计确认
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
布局: [布局类型]
重要模块: [N 个]
次要模块: [N 个]
关键元素: [元素列表]
交互: [N 条]
弹窗/浮层: [N 个，列出名称]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
以上描述是否正确？
```

确认后继续下一页面。

---

### Step 4：全部页面确认门禁

所有页面沟通完毕、全部写入 `02-page-specs.md` 后，向用户展示最终确认门禁（参见 AGENTS.md 确认门禁 2）。

用户确认后 → 推进到阶段 3。
