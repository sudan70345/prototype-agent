# 业务架构沟通 Skill — 阶段 1

## 输入

- 用户提供的需求描述（来自阶段 1 对话）

## 前置条件

- 已获取 `task_name` 并创建 `prototype/{task_name}_{YYYYMMDD}/` 目录

## 输出

- `prototype/{task_name}_{YYYYMMDD}/01-business-architecture.md`

## 工作流程

### Step 1：引导用户提供信息

使用 AGENTS.md 中的引导模板，逐步了解：
1. **端**：移动端 / Web 端 / 多端
2. **功能类型**：表单处理 / 数据分析 / 内容展示 / 混合
3. **需求背景**：业务场景、解决什么问题
4. **目标用户**：用户画像、使用频率
5. **关键页面**：每个页面的核心职责（解决什么问题、提供什么服务）
6. **页面流转**：从 A 到 B 的流程路径
7. **UI 风格**：有无参考、风格偏好、品牌规范

### Step 2：整理信息 → 输出业务架构文档

写入 `prototype/{task_name}_{YYYYMMDD}/01-business-architecture.md`，结构如下：

```markdown
# {任务名称} — 业务架构描述

## 1. 需求描述
- **端**: [移动端 / Web端]
- **功能类型**: [表单处理 / 数据分析 / 内容展示 / 混合]
- **背景**: [业务背景]
- **目标用户**: [用户画像]

## 2. UI 约束描述
- **风格**: [风格描述]
- **参考**: [参考产品/链接]
- **品牌规范**: [如有]
- **色彩倾向**: [如有]

## 3. 页面拆解
### 页面1: [名称]
- **核心职责**: [解决什么问题、提供什么服务]
- **定位**: [核心页面 / 辅助页面]

### 页面2: [名称]
- **核心职责**: […]
- **定位**: [核心页面 / 辅助页面]

...（共 N 页）

## 4. 页面流转
[流程图或表格描述]
| 来源页面 | 触发操作 | 目标页面 | 说明 |
|---------|---------|---------|------|
| 首页 | 点击商品 | 详情页 | 跳转 |
| ... | ... | ... | ... |
```

### Step 3：展示给用户确认

将文档摘要展示给用户（参见 AGENTS.md 确认门禁 1）。

- 用户提出修改 → 更新文档，重新确认
- 用户确认 → 推进到阶段 2
