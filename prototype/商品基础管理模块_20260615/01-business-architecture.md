# 商品基础管理模块 — 业务架构描述

## 1. 需求描述

- **端**: Web 端（桌面端）
- **功能类型**: 混合（表单处理 + 数据列表展示 + 弹窗交互）
- **背景**: 进销存系统的基础数据管理模块，用于维护商品元数据。采购人员通过此模块管理商品基础信息、商品分类层级和品牌信息，为后续的采购、入库、销售等业务提供基础数据支撑。
- **目标用户**: 采购人员，低频使用（有商品需要维护时使用），需要清晰、高效的交互体验。

## 2. UI 约束描述

- **风格**: 简洁清爽型，体现科技含量
- **设计体系**: 参考 Ant Design 组件风格进行设计
- **色彩倾向**: 蓝绿/青色系（现代科技感），非传统蓝色
- **品牌规范**: 无特定企业规范，使用通用后台布局
- **布局**: 左侧菜单 + 右侧内容区的经典后台管理布局

## 3. 页面拆解

### 页面1: 商品基础信息管理（核心页面）
- **核心职责**: 商品的增删改查管理，维护商品名称、规格、价格、库存等核心属性
- **定位**: 核心页面（主要使用页面）
- **特殊交互**:
  - 创建/编辑/查看商品 → 大弹窗（表单型子页面）
  - 上架/下架/删除操作 → 小弹窗二次确认
  - 库存维护 → 小弹窗选择增/减库存 + 输入变化量 + 确认
  - 类目和品牌数据通过接口抓取填充到商品表单中

### 页面2: 商品类目管理（辅助页面）
- **核心职责**: 维护商品分类层级结构（如一级类目/二级类目/三级类目）
- **定位**: 辅助页面（低频维护）
- **特殊交互**:
  - 创建/修改类目 → 小弹窗（表单型）
  - 删除类目节点 → 二次确认弹窗 / 删除异常提醒弹窗

### 页面3: 商品品牌管理（辅助页面）
- **核心职责**: 维护品牌信息
- **定位**: 辅助页面（低频维护）
- **特殊交互**:
  - 创建/修改品牌 → 小弹窗（表单型）
  - 删除品牌 → 二次确认弹窗

## 4. 页面流转

三个页面平级，通过左侧菜单独立访问，无直接页面间跳转。

| 来源页面 | 触发操作 | 目标页面 | 说明 |
|---------|---------|---------|------|
| 左侧导航菜单 | 点击菜单项 | 各页面 | 通过公共菜单切换 |
| 商品基础信息管理 | 创建/编辑/查看 | 大弹窗(当前页) | 弹窗内展示商品表单 |
| 商品基础信息管理 | 上架/下架/删除 | 小弹窗(当前页) | 二次确认 |
| 商品基础信息管理 | 库存维护 | 小弹窗(当前页) | 选择增/减 + 输入数量 |
| 商品类目管理 | 创建/修改 | 小弹窗(当前页) | 弹窗内展示类目表单 |
| 商品类目管理 | 删除 | 小弹窗(当前页) | 二次确认 / 异常提醒 |
| 商品品牌管理 | 创建/修改 | 小弹窗(当前页) | 弹窗内展示品牌表单 |
| 商品品牌管理 | 删除 | 小弹窗(当前页) | 二次确认 |
