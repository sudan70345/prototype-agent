# 供应商管理模块 — 业务架构描述

## 1. 需求描述

- **端**: Web 端（桌面端）
- **功能类型**: 表单处理（列表展示 + 弹窗交互）
- **背景**: 平台运营管理人员管理入驻供应商的基础信息，支持供应商的新建、编辑、删除等操作
- **目标用户**: 平台运营管理人员，日常高频使用

## 2. UI 约束描述

- **风格**: 简洁清爽型，体现科技含量
- **设计体系**: 参考 Ant Design 组件风格进行设计
- **色彩倾向**: 蓝绿/青色系（现代科技感），非传统蓝色
- **品牌规范**: 无特定企业规范，使用通用后台布局
- **布局**: 左侧菜单 + 右侧内容区的经典后台管理布局

## 3. 页面拆解

### 页面1: 供应商信息管理（核心页面）
- **核心职责**: 供应商的增删改查管理，维护供应商编码、名称、联系人、联系电话、供应分类、等级、状态等信息
- **定位**: 唯一核心页面
- **特殊交互**:
  - 新建/编辑供应商 → 大弹窗（表单型）
  - 删除供应商 → 小弹窗二次确认
  - 供应分类字段为多选，数据来源为字典

## 4. 页面流转

只有一个页面，通过左侧菜单独立访问。

| 触发操作 | 目标 | 说明 |
|---------|------|------|
| 左侧导航菜单 | 供应商信息管理页面 | 通过公共菜单访问 |
| 新建按钮 | 大弹窗（表单） | 供应商表单，含多选调取字典分类 |
| 编辑链接（操作列） | 大弹窗（表单） | 预填供应商数据 |
| 删除链接（操作列） | 小弹窗（确认） | 二次确认 |
